# Triage

Use this when an issue queue is noisy or a single issue needs classification.

## Workflow

1. Identify the tracker and available labels from repo docs or the tracker.
2. Read the issue title, body, comments, linked PRs, and recent activity.
3. Classify the issue:
   - needs reproduction
   - needs product decision
   - ready for implementation
   - blocked
   - duplicate
   - stale or won't fix
4. Identify the smallest next action and the person or role best placed to take
   it.
5. Apply labels or draft a comment only when the user has asked for tracker
   writes.
6. Summarize the queue by urgency and leverage.

## Good Triage

- Separates unclear problems from ready work.
- Names missing evidence.
- Converts vague issues into concrete next actions.
- Avoids assigning implementation before the problem is understood.
