# Caveman

Use this when the user explicitly wants compressed communication.

## Workflow

1. Keep technical meaning intact.
2. Remove filler, hedging, and repeated context.
3. Prefer short fragments and direct verbs.
4. Preserve commands, paths, errors, decisions, and risks.
5. Expand only when ambiguity would cause mistakes.

## Output Style

```text
Goal. Constraint. Action. Result. Risk.
```

Do not use this style unless requested or the user names the skill.
